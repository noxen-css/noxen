The framework that knows what things are.

Bootstrap tells you what things look like. Noxen knows what things are.

Why Noxen?

Every CSS framework since 2011 makes you describe how things look. Bootstrap gives you class names for appearance. Tailwind gives you hundreds of utility classes. You write the visual properties manually.

Noxen flips this completely. You describe what something is — a card, a button, a section — and Noxen applies the entire design system automatically. The result is markup that reads like English, themes that switch in 1ms, and a design system that scales from solo project to enterprise.

Bootstrap
<div class="card shadow rounded p-3 mb-3 bg-white border">
Tailwind
<div class="bg-white rounded-xl shadow-md p-6 flex flex-col gap-3 border border-gray-100 hover:shadow-lg transition-all">
✓ Noxen
<div nx="card" variant="raised">

<!-- Noxen CSS --> <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/noxen-css@2/dist/noxen.min.css"> <!-- Noxen JS (themes, interactions, AI palette) --> <script src="https://cdn.jsdelivr.net/npm/noxen-css@2/dist/noxen.min.js"></script>

npm install noxen-css
// In your JS/TS entry file: import 'noxen-css' // With bundlers (Vite, Webpack, Parcel): import 'noxen-css/dist/noxen.min.css' import 'noxen-css/dist/noxen.min.js'

Accessibility (Auto-ARIA)

WCAG 2.1 AA is the default, not an afterthought.

Every Noxen component automatically injects the correct ARIA roles, keyboard navigation, and focus management. You get accessibility for free — without any extra attributes.

What's Automatic

Buttons

role="button" + tabindex + Enter/Space keydown

Toggles

role="switch" + aria-checked state management

Alerts

role="alert" or "status" + aria-live regions

Accordions

aria-expanded + aria-controls linkage

Tabs

role="tablist/tab" + roving tabindex

Focus

:focus-visible with accent color ring on all elements

Noxen.theme.set('neon') // Switch theme Noxen.theme.get() // Get current theme name Noxen.theme.list() // List all available themes Noxen.theme.create('name', {}) // Create custom theme Noxen.theme.remove('name') // Remove custom theme

Token System

Every design decision is a CSS custom property. Everything is overridable.

Tokens follow the pattern: --nx-[category]-[role]-[modifier]

Category Example Tokens Description
color --nx-color-accent, --nx-color-bg, --nx-color-surface-2 All colors — backgrounds, text, accent, semantic
space --nx-space-1 through --nx-space-10 4px base: 4, 8, 12, 16, 24, 32, 48, 64, 96, 128px
text --nx-text-xs through --nx-text-6xl Type scale. Also --nx-fluid-sm through --nx-fluid-hero
radius --nx-radius-xs through --nx-radius-full 4, 6, 10, 16, 24, 32, 9999px
shadow --nx-shadow-xs through --nx-shadow-xl, --nx-shadow-glow 5 elevation levels + glow effect
motion --nx-ease-spring, --nx-duration-base Easing curves and duration tokens
font --nx-font-display, --nx-font-body, --nx-font-mono Font family system
z-index --nx-z-dropdown, --nx-z-modal, --nx-z-toast Named z-index scale
brand --nx-brand-primary, --nx-brand-secondary Brand override zone — customize these
