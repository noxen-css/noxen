The framework that knows what things are.

Bootstrap tells you what things look like. Noxen knows what things are.

Why Noxen?

Every CSS framework since 2011 makes you describe how things look. Bootstrap gives you class names for appearance. Tailwind gives you hundreds of utility classes. You write the visual properties manually.

Noxen flips this completely. You describe what something is — a card, a button, a section — and Noxen applies the entire design system automatically. The result is markup that reads like English, themes that switch in 1ms, and a design system that scales from solo project to enterprise.

Bootstrap
<div class="card shadow rounded p-3 mb-3 bg-white border">
Tailwind
<div class="bg-white rounded-xl shadow-md p-6 flex flex-col gap-3 border border-gray-100 hover:shadow-lg transition-all">
✓ Noxen
<div nx="card" variant="raised">

Installation

Three ways to use Noxen. Pick what fits your project.

Option 1 — CDN (Recommended for getting started)

Copy these two lines into your HTML <head>. No install, no build step, no config.

<!-- Noxen CSS --> <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/noxen-css@2/dist/noxen.min.css"> <!-- Noxen JS (themes, interactions, AI palette) --> <script src="https://cdn.jsdelivr.net/npm/noxen-css@2/dist/noxen.min.js"></script>

Option 2 — NPM

npm install noxen-css
// In your JS/TS entry file: import 'noxen-css' // With bundlers (Vite, Webpack, Parcel): import 'noxen-css/dist/noxen.min.css' import 'noxen-css/dist/noxen.min.js'

Option 3 — Download

Download the ZIP from GitHub and include the dist/ files directly in your project.

All three methods give you the same full framework — all 7 themes, all 25+ components, the Antigravity Engine, and the full JS API.

Basic HTML Template

<!DOCTYPE html> <html lang="en" data-nx-theme="void"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/noxen-css@2/dist/noxen.min.css"> </head> <body> <div nx="section" size="hero"> <div nx="stack" align="center"> <h1>Hello Noxen</h1> <button nx="btn" size="lg">Get Started</button> </div> </div> <script src="https://cdn.jsdelivr.net/npm/noxen-css@2/dist/noxen.min.js"></script> </body> </html>

// Create a brand theme from token overrides Noxen.theme.create('mybrand', { '--nx-color-accent': '#FF3366', '--nx-color-accent-lo': 'rgba(255,51,102,0.08)', '--nx-color-accent-md': 'rgba(255,51,102,0.20)', '--nx-color-bg': '#0a0005', }) Noxen.theme.set('mybrand')

Accessibility (Auto-ARIA)

WCAG 2.1 AA is the default, not an afterthought.

Every Noxen component automatically injects the correct ARIA roles, keyboard navigation, and focus management. You get accessibility for free — without any extra attributes.

What's Automatic

Buttons

role="button" + tabindex + Enter/Space keydown

Toggles

role="switch" + aria-checked state management

Alerts

role="alert" or "status" + aria-live regions

Accordions

aria-expanded + aria-controls linkage

Tabs

role="tablist/tab" + roving tabindex

Focus

:focus-visible with accent color ring on all elements

A11y Audit

// Run a full WCAG audit on the current page const result = Noxen.a11y.audit() // → { issues: [...], errors: 0, warnings: 2 } // High contrast mode document.documentElement.setAttribute('data-nx-a11y', 'high-contrast')

Reduced Motion

All animations and transitions are automatically disabled when the user has prefers-reduced-motion: reduce set in their OS. No extra work needed.

JavaScript API

Full programmatic control over every aspect of Noxen.

Noxen.init(config)

Noxen.init({ theme: 'carbon', // Set initial theme tokens: { // Override tokens on init '--nx-color-accent': '#FF3366' }, a11y: true, // Enable auto-ARIA (default: true) debug: true // Enable debug logging })

Theme API

Noxen.theme.set('neon') // Switch theme Noxen.theme.get() // Get current theme name Noxen.theme.list() // List all available themes Noxen.theme.create('name', {}) // Create custom theme Noxen.theme.remove('name') // Remove custom theme

Token API

Noxen.tokens.set('--nx-color-accent', '#FF3366') Noxen.tokens.setMany({ '--nx-color-accent': '#FF3366', ... }) Noxen.tokens.get('--nx-color-accent') // → '#FF3366' Noxen.tokens.reset('--nx-color-accent') // Reset to theme default Noxen.tokens.resetAll() // Reset all overrides Noxen.tokens.export('css') // Export in any format

Antigravity API

const el = document.querySelector('#my-card') Noxen.ag.mirror(el) // Apply mirror Noxen.ag.reflect(el) // Apply reflection Noxen.ag.restore(el) // Remove antigravity Noxen.ag.toggle(el) // Toggle mirror on/off Noxen.ag.enter(el, 'rise') // Trigger entry animation

Events

Noxen.on('ready', (data) => console.log('Noxen ready', data.version)) Noxen.on('theme', (data) => console.log('Theme changed:', data.theme)) Noxen.on('token', (data) => console.log('Token set:', data.key, data.value)) Noxen.on('a11y', (data) => console.log('Audit done:', data.errors, 'errors'))

Token System

Every design decision is a CSS custom property. Everything is overridable.

Tokens follow the pattern: --nx-[category]-[role]-[modifier]

CategoryExample TokensDescription
color--nx-color-accent, --nx-color-bg, --nx-color-surface-2All colors — backgrounds, text, accent, semantic
space--nx-space-1 through --nx-space-104px base: 4, 8, 12, 16, 24, 32, 48, 64, 96, 128px
text--nx-text-xs through --nx-text-6xlType scale. Also --nx-fluid-sm through --nx-fluid-hero
radius--nx-radius-xs through --nx-radius-full4, 6, 10, 16, 24, 32, 9999px
shadow--nx-shadow-xs through --nx-shadow-xl, --nx-shadow-glow5 elevation levels + glow effect
motion--nx-ease-spring, --nx-duration-baseEasing curves and duration tokens
font--nx-font-display, --nx-font-body, --nx-font-monoFont family system
